# Phase 3: Vector ALU & RNS Manager

## Цель
Превратить модуль NTT в полноценный криптопроцессор, способный выполнять гомоморфные операции (Add, Mult) над длинными векторами в RNS-представлении.

## План "10 Дней" (Предварительный)

### Этап 1: Векторная Арифметика (ALU)
* **День 1:** Модуль `mod_vec_add.v` и `mod_vec_mult.v` (Покомпонентные операции).
* **День 2:** Интеграция ALU в `ntt_engine` (расширение FSM: NTT -> ALU -> INTT).
* **День 3:** Тест гомоморфного сложения и умножения (Python SDK -> HW).

### Этап 2: RNS (Residue Number System)
* **День 4:** Архитектура Multi-Core или Time-Multiplexing. (Как обрабатывать 3-4 простых числа Q1, Q2, Q3?).
* **День 5:** Контекстный Менеджер в C++ (управление наборами ключей и модулей).
* **День 6:** Загрузка нескольких наборов Twiddle Factors (Bank Switching).

### Этап 3: Ключевые Операции
* **День 7:** Генерация шума (Gaussian Sampler) - если решим делать шифрование в железе.
* **День 8:** Relin Key Support (Базовая поддержка для умножения).
* **День 9:** Интеграция с высокоуровневым API (подобие SEAL API).
* **День 10:** Демонстрация: Вычисление полинома `x^2 + x + 1` над зашифрованными данными.

## Критерии Успеха
1. BFV Homomorphic Addition выполняется в железе.
2. BFV Homomorphic Multiplication выполняется в железе.
3. Поддержка минимум 2-х RNS модулей (Q1, Q2).
